mac通过hexo+github搭建个人博客

hexo github
Markdown
发布日期:   2021-09-27

引言

Hexo 是一个静态博客框架，基于 Node.js，将 Markdown 文章通过渲染引擎，生成一个静态网页，再结合 Git 命令（ssh），Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

1. 申请github账号并创建仓库

访问github，如果没有github账号新建一个，有github账号的话登录github账号。新建一个仓库，并将仓库的名字命名为：username.github.io, username是github账号的名字(owner)。

2. mac上配置博客环境

2.1. 安装node.js和hexo

进入链接，选择长期稳定版(LST)，点击下载对应操作系统的版本, 安装包一路next下去就行。

安装完成以后输入一下指令进行检查：

node -v
npm -v

通过输入以下指令安装hexo，

sudo npm install -g hexo

2.2. 安装git

没有安装homebrew的，先安装homebrew，

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装git

brew install git

验证是否有效安装，

Git --version

若是安装无误，配置Git，设置username和email，

git config --global user.name "linghuchong"
git config --global user.email "52013149836@qq.com"

通过终端命令行生成ssh密钥，

ssh-keygen -t rsa -C "52013149836@qq.com"

密钥生成后确认是否成功，

cd ~/.ssh
ls

如果密钥生成的话会看到列表里面包含id_rsa(私钥), id_rsa.pub(公钥)

打开id_rsa.pub，

cat ~/.ssh/id_rsa.pub

复制公钥，

在github中添加公钥，

3. 博客基本设置

为了查找方便，在桌面上新建了一个文件夹myblog，打开终端，

cd desktop
cd myblog

hexo框架的安装，

npm install -g hexo-cli

初始化文件夹

hexo init

安装博客所需要的依赖文件，

npm install

等待运行完成时，该文件夹中就多了许多文件，hexo的基本框架搭建完毕。

4. 搭建第一个博客

4.1 本地运行hexo博客

打开终端，运行以下指令，

hexo clean && hexo g
hexo s

复制窗口中出现的链接，可以看到一个网页，说明hexo博客已在本地成功运行。

4.2 将hexo博客部署到github上去

首先需要安装发布的插件，打开终端，cd跳转到创建的博客目录下，在这里也就是myblog下，执行，

sudo npm install hexo-deployer-git --save

在博客目录中有一个_config.yml文件，是博客的配置文件。通过文本编辑器(vscode, sublime)打开_config.yml文件，下拉到文件最底部，有一个deploy，添加以下语句，username是自己github的用户名。

deploy:
  type: git
  repository: git@github.com:username/username.github.io.git
  branch: master

终端下执行如下命令，生成页面，并发布到github pages上，

hexo clean && hexo g
hexo d

上传完成后，在浏览器中打开https://.github.io查看上传的网页，如果页面变成了本地调试的样子，代表部署成功。

4.3 基本设置

在myblog文件夹下有一个文件_config.yml, 同时在myblog/themes/hexo-theme-matery文件夹下也有一个config.yml文件，前者称为站点配置文件，后者称为主题配置文件。

4.3.1 更改主题

hexo博客自带的主题不太好看，因此需要我们自己更改为一个喜欢的主题。这里我选择了matery，可以通过链接下载，然后将hexo-theme-matery文件夹复制到myblog/themes文件夹中。也可以在myblog目录下使用git进行下载，

git clone https://github.com/blinkfox/hexo-theme-matery themes/matery # 稳定版
git clone -b develop https://github.com/blinkfox/hexo-theme-matery themes/matery # 最新版

主题下载完毕后，打开myblog中的站点配置文件_config.yml，修改theme值为hexo-theme-matery-master，

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: hexo-theme-matery-master

站点配置文件_config.yml中的其余设置根据需要进行更改，例如：

  • 语言选择：如果为想要使用中文，则在language:后添加值zh-CN，如果不修改，默认为英语；
  • 站点配置文件有个per_page属性，建议修改为6的倍数，比如12，18，这样网站在适应设备时，有较好的显示效果；
  • 网址修改：url：的值修改为自己的网址名，如https://username.github.io,如果有域名，修改为自己的域名即可。
4.3.1 新建标签tags页面

tags页面是用来显示所有标签的页面，如果在你的博客的source目录下还没有tags/index.md文件，就需要新建一个，命令如下，

hexo new page "tags"

编辑你刚刚新建的页面文件 /source/tags/index.md，至少需要以下内容：

title: tags
date: 2021-09-28 18:23:38
type: "tags"
layout: "tags"
4.3.2 新建分类categories页面

categories 页是用来展示所有分类的页面，如果在你的博客 source 目录下还没有 categories/index.md 文件，那么你就需要新建一个，命令如下：

hexo new page "categories"

编辑你刚刚新建的页面文件 /source/categories/index.md，至少需要以下内容：

title: categories
date: 2021-09-28 17:25:30
type: "categories"
layout: "categories"
4.3.3 新建关于我about页面

编辑你刚刚新建的页面文件 /source/about/index.md，至少需要以下内容：

title: about
date: 2021-09-28 17:25:30
type: "about"
layout: "about"

在主题的 _config.yml 文件中，默认支持 QQ、GitHub 和邮箱等的配置，

4.4 写博客

打开终端，cd进myblog文件夹，新建博客，

hexo new “博客名称”

此时，会在source/_posts下产生一个博客名称.md文件和一个以博客名称命名的文件夹，可以将文件夹删除。
